# Manumation Principles

These principles guide every product decision, agent behavior, workflow, and pull request in Donna-V7.

## 1. Humans keep judgment

The system may research, route, summarize, critique, recommend, draft, and execute approved actions.

The human remains responsible for vision, ethics, relationship judgment, and final approval on meaningful risk.

## 2. AI improves thinking before it improves speed

Speed is useful only after the system understands the objective, context, risks, assumptions, and approval requirements.

A fast wrong answer is still wrong. Humans keep inventing ways to confuse velocity with wisdom, so the software must resist that nonsense.

## 3. Memory is more valuable than response

Manumation does not exist to generate more messages.

It exists to remember the details that make communication, strategy, service, and execution more thoughtful.

## 4. Every meaningful action creates or updates a Decision Object

Chats disappear into the swamp.

Decision Objects preserve what mattered:

- objective
- evidence
- assumptions
- options
- risks
- recommendation
- confidence
- approval
- outcome
- lesson learned

## 5. Donna governs before she assists

Donna is not merely a chatbot.

Donna is the governance interface for the Intelligence Layer. She routes work, checks context, questions assumptions, compares agent outputs, watches risk, and decides when the human must approve.

## 6. Confidence must be earned

The system must never present false certainty.

Confidence scores must be tied to evidence quality, retrieved context, contradiction checks, risk level, and missing information.

## 7. Multi-tenant trust is non-negotiable

Client A must never see Client B’s data.

Every table, query, API route, background job, vector search, and AI prompt must respect tenant isolation.

## 8. The loop is the product

The Manumation Evolution Loop is not decoration.

It is the core product behavior.

Any feature that bypasses the loop must justify why bypassing quality control is safer than using it.

## 9. Prompts are implementation details

Prompts matter, but they are not the moat.

The moat is cognitive architecture, governance, memory, decision history, and outcome learning.

## 10. Release only when useful

A feature is not done because it works.

It is done when it improves judgment, preserves context, reduces unnecessary work, protects trust, and can be explained to a tired business owner in plain English.
